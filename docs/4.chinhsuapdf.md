# 4. Quy Trình Chỉnh Sửa & Xuất Bản PDF (pdf-lib)

Tài liệu này giải thích chi tiết cách ứng dụng tương tác cấu trúc và xuất bản file PDF bằng thư viện **`pdf-lib`**. 

---

## 1. Nguyên lý chỉnh sửa PDF không mất chất lượng (Non-destructive Editing)

Nếu chúng ta convert toàn bộ trang PDF thành ảnh để chỉnh sửa rồi đóng gói lại thành PDF, chất lượng văn bản sẽ bị vỡ (rasterization), người dùng không thể tìm kiếm từ khóa (Searchable) hay bôi đen copy chữ được nữa.

**Giải pháp của chúng ta:**
1. **Quản lý cấu trúc trang ảo**: Trong quá trình chỉnh sửa, file PDF gốc không hề bị thay đổi. Chúng ta chỉ quản lý danh sách trang ảo trong React State dưới dạng mảng cấu hình (`pages`).
2. **Áp dụng thay đổi khi xuất bản (Export)**: Khi người dùng nhấn nút **Tải file về**, ứng dụng sẽ tạo mới một tài liệu trống trong bộ nhớ, sao chép các trang được chọn từ PDF gốc qua, thiết lập góc xoay mới, vẽ các hình chữ nhật đè (Vector) hoặc ảnh vá đè (Inpaint image) lên đúng vị trí, rồi xuất file kết quả. Điều này giúp bảo toàn 100% định dạng, phông chữ và tính chất vector của file PDF gốc.

---

## 2. Các thao tác quản lý trang (Page Operations)

### 1. Thêm trang trống mới (Add Blank Page)
Khi người dùng thêm trang mới, chúng ta tạo một cấu hình trang ảo với thuộc tính `originalIndex: null`. Khi xuất bản:
```javascript
// Tạo một trang trắng tinh với kích thước tiêu chuẩn A4 (595 x 841 points)
const w = pageConfig.width || 595.276;
const h = pageConfig.height || 841.890;
dstPage = dstDoc.addPage([w, h]);
```

### 2. Xóa trang (Delete Page)
Chúng ta lọc trang đó ra khỏi mảng trạng thái `pages` của React. Khi đóng gói xuất bản, trang bị xóa sẽ không được sao chép từ tài liệu gốc sang tài liệu mới, do đó biến mất hoàn toàn.

### 3. Thay đổi thứ tự trang (Reorder Pages)
Sắp xếp lại các phần tử trong mảng trạng thái `pages`. Khi xuất bản, chúng ta duyệt mảng `pages` từ đầu đến cuối và sao chép các trang từ tài liệu gốc theo đúng thứ tự mới này:
```javascript
const [copiedPage] = await dstDoc.copyPages(srcDoc, [pageConfig.originalIndex - 1]);
dstPage = dstDoc.addPage(copiedPage);
```

### 4. Nhân bản trang (Duplicate Page)
Bằng cách sao chép cấu hình của trang (bao gồm cả các nét vẽ cũ nhưng được cấp ID mới) và chèn ngay phía sau. Khi xuất bản, hàm `copyPages` của `pdf-lib` có thể sao chép cùng một trang của tài liệu gốc nhiều lần mà không gặp bất kỳ lỗi nào.

### 5. Xoay trang (Rotate Page)
Mỗi trang có góc xoay riêng (`rotation: 0 | 90 | 180 | 270`). Khi đóng gói, ta gọi:
```javascript
dstPage.setRotation(degrees(pageConfig.rotation));
```

## 3. Vẽ nét chỉnh sửa đè lên trang PDF gốc

Quá trình vẽ các nét sửa đổi diễn ra trong vòng lặp đóng gói từng trang:

### 1. Vẽ nét Highlight
Vẽ một hình chữ nhật bán trong suốt (opacity = 0.4) đè lên vị trí văn bản:
```javascript
const { r, g, b } = hexToRgb(color || '#ffeb3b');
dstPage.drawRectangle({
  x, y,
  width: w,
  height: h,
  color: rgb(r, g, b),
  opacity: 0.4, // Độ trong suốt cho nét highlight
});
```

### 2. Vẽ nét Xóa che màu (Solid Cover)
Vẽ một hình chữ nhật đặc (opacity = 1.0) đè lên logo. Màu sắc được chọn tiệp với màu nền của PDF (mặc định là màu trắng):
```javascript
const { r, g, b } = hexToRgb(color || '#ffffff');
dstPage.drawRectangle({
  x, y,
  width: w,
  height: h,
  color: rgb(r, g, b),
  opacity: 1.0,
});
```

### 3. Vẽ nét Xóa hòa nhập (Inpaint Overlay)
Đối với inpaint, nét chỉnh sửa chứa một ảnh patch dạng Base64 PNG được sinh ra từ giải thuật Laplacian. Chúng ta tiến hành nhúng và vẽ ảnh này đè lên vùng logo:
1. **Chuyển Base64 thành mảng byte `Uint8Array`**:
   ```javascript
   export function base64ToUint8Array(base64Str) {
     const base64Data = base64Str.split(',')[1] || base64Str;
     const raw = window.atob(base64Data);
     const uint8Array = new Uint8Array(raw.length);
     for (let i = 0; i < raw.length; i++) {
       uint8Array[i] = raw.charCodeAt(i);
     }
     return uint8Array;
   }
   ```
2. **Nhúng ảnh PNG vào tài liệu PDF mới**:
   ```javascript
   const imgBytes = base64ToUint8Array(imageSrc);
   const embeddedImage = await dstDoc.embedPng(imgBytes);
   ```
3. **Vẽ ảnh đè lên tọa độ PDF**:
   ```javascript
   dstPage.drawImage(embeddedImage, {
     x, y,
     width: w,
     height: h,
   });
   ```

## 4. Tải file kết quả về máy người dùng (Download Workflow)

Sau khi hoàn tất vòng lặp vẽ đè toàn bộ các trang, ta chuyển tệp PDF đã lưu trong bộ nhớ thành dạng dữ liệu Blob để tải về:
```javascript
// 1. Lưu tài liệu thành mảng byte Uint8Array
const editedBytes = await dstDoc.save();

// 2. Tạo đối tượng Blob ứng với định dạng PDF
const blob = new Blob([editedBytes], { type: 'application/pdf' });
const url = URL.createObjectURL(blob);

// 3. Tạo thẻ link <a> ẩn và giả lập click để tải về
const link = document.createElement('a');
link.href = url;
link.download = 'edited_document.pdf';
document.body.appendChild(link);
link.click();

// 4. Giải phóng bộ nhớ tạm sau khi tải xong
document.body.removeChild(link);
URL.revokeObjectURL(url);
```
Quy trình này diễn ra hoàn toàn trên trình duyệt của máy khách (Client-side), không gửi bất kỳ dữ liệu nhạy cảm nào lên Internet, đảm bảo tính bảo mật tuyệt đối cho thông tin của người sử dụng.
